2️⃣ Smart Delete Protection

When you delete a category, the app intelligently handles existing transactions:

Step 1: Confirmation Dialog

Shows the category name and transaction count. Example: Delete "Restaurants"? (3 transactions)

Step 2: Transaction Handling

If the category has transactions, you choose:

  • Move to Uncategorized — All transactions stay in the app but lose their category assignment
  • Delete Transactions — Transactions are permanently removed with the category
Step 3: Cleanup

The category's budget is deleted. All calculations instantly update.

5️⃣ Budget vs Actual Spending (Correct Separation)

Key Principle: Budgets are targets. Only transactions represent real spending.

Example:

  • Restaurants budget: 200 KD (target)
  • Transactions in Restaurants: 65 KD (actual spending)
  • Remaining budget: 200 - 65 = 135 KD
  • Remaining balance: Income - totalSpent() (not affected by budget)

How it works:

  • getCatSpent(catId) — sums only transactions in that category
  • totalBudget() — sums budget targets across all categories
  • totalSpent() — sums all transactions (total real spending)
  • Remaining balance = Income - totalSpent() (not affected by budget amounts)

Dashboard shows:

  • ✅ Total Budget (sum of category targets)
  • ✅ Total Spent (sum of all transactions)
  • ✅ Remaining (Income - Total Spent)
  • ✅ Budget Used % (Total Spent / Total Budget)
  • ✅ Per-category breakdown: Budget | Spent | Remaining | Over/Under

9️⃣ Technical Details

Data Model: 
Categories (objects):
- id: string (unique)
- name: string (max 20 chars)
- icon: string (emoji)
- color: string (hex)
- budget: number (KD)
- isDefault: boolean (editable, not deletable)
- isHidden: boolean (for future feature)
- order: number (drag-drop ordering)

Transactions (objects):
- id: string (unique)
- amount: number (KD)
- category: string or null (uncategorized)
- date: string (YYYY-MM-DD)
- notes: string (max 100 chars)

Settings:
- income: number (KD)
- Storage key: "miza_data_v4"
- Storage: localStorage only (no server, no cloud)
Performance:
  • Service worker caches HTML + manifest (offline support)
  • Cache version: miza-v3 (forces fresh load on deployment)
  • All calculations are O(n) — even with 1000 transactions, updates are instant
Validation:
  • Duplicate category names are prevented
  • Budget amounts must be non-negative
  • Transaction amounts must be positive
  • Category names limited to 20 characters
